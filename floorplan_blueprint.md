# 🏠 AI Floor Plan Generator — Full Product Blueprint

> Hand this document to any AI code editor (Gemini, Cursor, GitHub Copilot, etc.)

---

## 1. PRODUCT SUMMARY

A web app where a user inputs their plot dimensions and optional preferences,
and instantly sees a proportionally accurate, labeled floor plan of how a house
could be built on that plot — generated by AI and rendered as a clean SVG diagram.

**NOT** a construction tool. This is a **quick visualization / first-look tool.**

## 4. DATA TYPES (`/lib/types.ts`)

```typescript
export interface PlotInputs {
  // Required
  lengthFt: number;          // Plot length in feet
  breadthFt: number;         // Plot breadth in feet

  // Optional
  frontRoad?: "North" | "South" | "East" | "West";
  orientation?: "North" | "South" | "East" | "West";
  floors?: number;           // 1, 2, or 3
  bedrooms?: number;         // 1–5
  bathrooms?: number;        // 1–4
  parking?: boolean;
  garden?: boolean;
  poojaRoom?: boolean;
  servantQuarters?: boolean;
  kitchenType?: "open" | "closed";
  vastu?: boolean;
  style?: "modern" | "traditional" | "minimal";
  familyType?: "nuclear" | "joint";
}

export interface Room {
  id: string;                // e.g. "bedroom-1", "kitchen", "staircase"
  label: string;             // Display name e.g. "Master Bedroom"
  xPercent: number;          // Left position as % of plot width (0–100)
  yPercent: number;          // Top position as % of plot height (0–100)
  widthPercent: number;      // Width as % of plot width
  heightPercent: number;     // Height as % of plot height
  widthFt: number;           // Actual width in feet
  heightFt: number;          // Actual height in feet
  color?: string;            // Optional fill color for room type
}

export interface FloorPlan {
  floor: number;             // 0 = ground, 1 = first, 2 = second
  plotLengthFt: number;
  plotBreadthFt: number;
  rooms: Room[];
  staircasePosition?: {      // Saved from ground floor, reused for upper floors
    xPercent: number;
    yPercent: number;
    widthPercent: number;
    heightPercent: number;
  };
  notes?: string;            // AI design notes e.g. "Vastu applied: kitchen faces East"
}
```

## 5. CLAUDE API PROMPT (`/lib/prompt-builder.ts`)

### System Prompt (fixed):

```
You are an Indian residential floor plan layout expert.
Your job is to generate a JSON floor plan layout for a house plot.
You must return ONLY valid JSON — no markdown, no explanation, no extra text.
The JSON must exactly match the FloorPlan interface provided.

Rules:
- All room positions and sizes must be in PERCENTAGES of the total plot area (0–100).
- Rooms must NOT overlap each other.
- All rooms combined must fit within the plot boundaries (total area roughly 60–75% of plot, leaving space for walls and setbacks).
- Every room must have widthFt and heightFt as realistic Indian room sizes in feet.
- Always include a staircase room if floors > 1, positioned realistically.
- Apply Vastu principles if vastu: true (kitchen East/South-East, master bedroom South-West, pooja room North-East, main entrance North/East).
- If front road direction is given, place the main entrance on that side.
- Label rooms clearly in English (e.g. "Master Bedroom", "Kitchen", "Drawing Room", "Staircase").
- Include a small passage/lobby near the entrance.
- For ground floor: generate full layout.
- For upper floors: staircase position will be provided — keep it EXACTLY the same, generate the rest.
```

### User Prompt (dynamic, built from PlotInputs):

```
Generate a ground floor plan for this plot:
- Plot size: {lengthFt} ft x {breadthFt} ft
- Front road facing: {frontRoad or "Not specified, use best judgement"}
- Orientation: {orientation or "Not specified"}
- Family type: {familyType or "Nuclear"}
- Bedrooms required: {bedrooms or "AI decides based on plot size"}
- Bathrooms required: {bathrooms or "AI decides"}
- Parking required: {parking}
- Garden/green space: {garden}
- Pooja room: {poojaRoom}
- Servant quarters: {servantQuarters}
- Kitchen type: {kitchenType or "closed"}
- Apply Vastu: {vastu}
- Style preference: {style or "modern"}

Return ONLY the JSON object matching the FloorPlan interface. No explanation.
```

### Upper Floor Prompt (different endpoint):

```
Generate floor {floorNumber} plan for this same plot.
Plot size: {lengthFt} ft x {breadthFt} ft.
The staircase is FIXED at this position and must not move:
{JSON.stringify(staircasePosition)}

Generate bedrooms, bathrooms, and open terrace/balcony as appropriate for an upper floor.
Apply same Vastu and style rules as ground floor.
Return ONLY the JSON object matching the FloorPlan interface.
```

## 6. API ROUTES

### `/app/api/generate-floor-plan/route.ts`

```typescript
import Anthropic from "@anthropic-ai/sdk";
import { buildGroundFloorPrompt } from "@/lib/prompt-builder";
import { PlotInputs, FloorPlan } from "@/lib/types";

export async function POST(req: Request) {
  const inputs: PlotInputs = await req.json();

  const client = new Anthropic();

  const message = await client.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 2000,
    system: SYSTEM_PROMPT, // from prompt-builder.ts
    messages: [
      { role: "user", content: buildGroundFloorPrompt(inputs) }
    ]
  });

  const rawText = message.content[0].type === "text" ? message.content[0].text : "";

  try {
    const layout: FloorPlan = JSON.parse(rawText);
    return Response.json({ success: true, layout });
  } catch {
    return Response.json({ success: false, error: "Invalid JSON from AI", raw: rawText }, { status: 500 });
  }
}
```

### `/app/api/generate-upper-floor/route.ts`

Same structure but uses upper floor prompt, accepts `{ inputs, floorNumber, staircasePosition }`.

## 7. SVG RENDERER (`/components/FloorPlanCanvas.tsx`)

```
Logic:
1. Receive FloorPlan JSON as prop
2. Calculate canvas pixel size from plot aspect ratio
   - e.g. plot is 30ft x 40ft → canvas is 450px x 600px (scale: 15px per foot)
   - Scale formula: scale = Math.min(600 / lengthFt, 800 / breadthFt)
3. For each room in layout:
   - Convert xPercent/yPercent/widthPercent/heightPercent to pixels
   - Draw SVG <rect> with a stroke and semi-transparent fill
   - Draw room label centered inside the rect (font size auto-scales)
   - Draw dimension labels: "10ft x 12ft" below room name
4. Draw outer plot boundary as a bold rect
5. Show compass direction indicator (N/S/E/W) in corner
6. Show "Front Road →" label on the correct side

Room color coding (by room type):
- Bedroom: #E8F4FD (light blue)
- Kitchen: #FFF3E0 (light orange)
- Bathroom/WC: #E8F5E9 (light green)
- Drawing Room / Hall: #F3E5F5 (light purple)
- Staircase: #FFF9C4 (light yellow)
- Parking: #ECEFF1 (light grey)
- Garden: #DCEDC8 (light lime)
- Pooja Room: #FCE4EC (light pink)
- Passage/Lobby: #F5F5F5 (off white)
```

## 9. MULTI-FLOOR LOGIC

```
State managed in page.tsx:

const [floors, setFloors] = useState<FloorPlan[]>([]);
const [activeFloor, setActiveFloor] = useState(0);

Step 1: Generate ground floor → save to floors[0]
Step 2: Extract staircase room from floors[0].rooms
Step 3: When user clicks "Floor 1" tab (if floors > 1):
        → Call /api/generate-upper-floor with:
          { inputs, floorNumber: 1, staircasePosition: extracted from step 2 }
        → Save result to floors[1]
Step 4: Repeat for Floor 2 if needed

Each floor tab only calls the API once (cache the result in state).
```

## 10. LOADING & ERROR STATES

```
Loading state:
- Show animated placeholder SVG (grey pulsing blocks) while AI generates
- Message: "Designing your floor plan..." (2–4 seconds typically)

Error state:
- If Claude returns invalid JSON → show: "Generation failed, try again"
- Add retry button
- Log raw Claude output to console for debugging

Validation:
- Before rendering, check: all rooms have xPercent + widthPercent ≤ 100
- Check no two rooms overlap (simple bounding box check)
- If validation fails → trigger auto-retry (max 2 times) before showing error
```
